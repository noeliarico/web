1.1 Paradigmas de programación

Un paradigma de programación establece la forma en la que se estructura el código de una aplicación.

Es uno de los criterios utilizados para clasificar un lenguaje de programación, aunque muchos lenguajes dan soporte a más de un paradigma.

  1. Imperativo: Se enfoca en cómo realizar tareas mediante una secuencia de instrucciones.
  2. Declarativo: Describe qué debe hacerse sin especificar los pasos exactos para lograrlo.
  3. Funcional: Se basa en la evaluación de funciones y evita el cambio de estado o datos mutables.
  4. Paradigma Orientado a Objetos (POO): Organiza el código en objetos que combinan datos y comportamiento.
  5. Paradigma Concurrente y Paralelo: Ejecuta múltiples procesos o hilos simultáneamente para mejorar el rendimiento.

Otros paradigmas: Lógico, Reactivo, Basado en Componentes, Basado en Eventos, Paradigma Relacional…

Los paradigmas de programación pueden combinarse entre ellos.

2 Programación Orientada a Objetos

Fundamentos

La POO es un paradigma, es decir, una teoría que suministra la base y modelo para resolver problemas.

En concreto, permite resolver problemas que surgen en el desarrollo de grandes cantidades de código. Algunos de ellos:

  • Código muy largo: dificil de leer, depurar, mantener.
  • Si algo falla, todo se rompe: si todo se ejecuta secuencialmente en el momento en el que una de las líneas falla todo lo demás deja de ejecutarse.
  • Dificultad en la escritura colaborativa de código: si no está organizado, es difícil independizar las partes de código que debe escribir cada desarrollador.

Componentes y pilares

Componentes:

  • Clases
  • Propiedades
  • Métodos
  • Objetos

Pilares

  • Abstracción
  • Encapsulamiento
  • Polimorfismo
  • Herencia

La POO es un paradigma, o sea una teoría que suministra la base y modelo para resolver problemas.

Abstracción:

  • La idea de POO es poder dividir un problema en subproblemas más pequeños
  • Cada clase se encarga de uno de esos subproblemas

-Encapsulamiento - metodos dentro de clases a los que no acceden los demas -Abstracción - abstraer al usuario de ciertas implementaciones. por ej cuando creamos num aleatorios no sabemos como se hace eso. esto permite colaboración en programas grandes -Herencia - cuando creamos las clases hay mucho codigo que se repeite entonces se puede crear una estructura jerarquica para disminuir el copy/paste de código -Polimorfismo - modificar funcion especifica de una clase para que haga algo adicional o distinto

2.1 Clases y objetos

  • clase -> representación abstracta de un conjunto de objetos que comparten propiedades y comportamiento (del mismo tipo).
  • objeto -> cada una de las instancias de alguna clase. Se crean durante la ejecución del código ocupando espacio en memoria.

Clases:

  • Plantilla que permite crear un objeto.
  • Una clase es capaz de definir un objeto junto teniendo en cuenta sus:
    • propiedades (atributos)
    • funcionalidades (métodos)

Objetos

El objeto es la instancia de una clase, consta de:

  • Estado: representado por los valores atributos del objeto, que reflejan sus propiedades.
  • Comportamiento: representado por los métodos del objeto, que reflejan su respuesta a a otros objetos.
  • Identidad: cada objeto tiene un nombre único que le permite interactuar con otros objetos.

Podemos interactual con los objetos.

Guardamos en memoria un objeto. Que existe hasta que la variables deja de usarse y pasa el garbage collector.

2.11 Creación por copia

Sin embargo, podemos forzar que esta creación sea por copia usando el módulo copy, que implementa las funciones, copy() y deepcopy(), que sirven para hacer copias simples y recursivas, respectivamente.

  • La diferencia entre copy() y deepcopy() solo se nota cuando al crear copias de objetos compuestos.

  • Los objetos compuesto son aquellos en los que alguno de sus atributos es también un objeto (no simple).

  • copy() crea un nuevo objeto y luego coloca referencias que apuntan a los objetos que componen al original, sin duplicarlos

  • deepcopy() crea un nuevo objeto y luego crea, recursivamente, copias de los objetos que componen a éste.

from copy import copy

class Persona:

    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad = edad
    
    def presentarse(self):
        return f"Hola, me llamo {self.nombre} y tengo {self.edad} años."

persona1 = Persona("Ana", 20)
# en vez de hacer persona1 = persona2
persona2 = copy(persona1)
persona1.nombre = "Claudia"
print(persona2)

2.12 Tipos de métodos

  • Métodos de instancia: métodos de cada objeto
    • Toman el parámetro self como primer parámetro, representando la instancia del método. Pueden tener más parámetros.
  • Métodos estáticos: vinculados a la clase y no al objeto.
    • Se pueden utilizar sin instanciar. Se definen usando un decorador @staticmethod.
  • Métodos de clase:
    • Emplemos el decorador @classmethod
    • En lugar de tomar un self como primer parámetro, toman un cls que apunta a la clase y no a la instancia del objeto cuando se llama al metodo.
    • Como solo tiene acceso al cls, no pueden modificar el estado de la instancia del objeto (eso requeriría acceso a self).

Los nombres de lo métodos, al igual que los de las variables, van siempre en minúsculas. https://peps.python.org/pep-0008/#function-and-variable-names

Son funciones asociadas a los objetos de una clase que sirven para implementar las acciones que dichos objetos pueden llevar a cabo. 2.12.3 Se pueden mezcla métodos de distintos tipos en la misma clase

class Termometro:
    def __init__(self, temperatura, unidad="Celsius", marca="Genérica", rango_medicion=(-50, 150), precision=1):
        self.temperatura = temperatura
        self.unidad = unidad
        self.marca = marca
        self.rango_medicion = rango_medicion
        self.precision = precision
    
    # Método de instancia: Convierte la temperatura almacenada en la instancia a Fahrenheit (si es Celsius)
    def a_fahrenheit(self):
        if self.unidad == "Celsius":
            return round((self.temperatura * 9/5) + 32, self.precision)
        return self.temperatura  # Si ya está en Fahrenheit, no hace conversión

    # Método de instancia: Convierte la temperatura almacenada en la instancia a Celsius (si es Fahrenheit)
    def a_celsius(self):
        if self.unidad == "Fahrenheit":
            return round((self.temperatura - 32) * 5/9, self.precision)
        return self.temperatura  # Si ya está en Celsius, no hace conversión

    # Método de instancia: Verifica si la temperatura está dentro del rango que soporta el termómetro
    def dentro_rango(self):
        return self.rango_medicion[0] <= self.temperatura <= self.rango_medicion[1]

    # Método estático: Convierte cualquier valor de Fahrenheit a Celsius sin depender de una instancia
    @staticmethod
    def fahrenheit_a_celsius(grados_fahrenheit):
        return (grados_fahrenheit - 32) * 5/9

    # Método estático: Convierte cualquier valor de Celsius a Fahrenheit sin depender de una instancia
    @staticmethod
    def celsius_a_fahrenheit(grados_celsius):
        return (grados_celsius * 9/5) + 32

# Uso de métodos de instancia
termometro = Termometro(25, "Celsius", "TermoX", (-50, 150), 2)  # Instancia con temperatura en Celsius
print(f"Marca: {termometro.marca}")
print(f"Temperatura en Fahrenheit: {termometro.a_fahrenheit()}°F")  # 25°C -> 77.00°F
print(f"¿Está dentro del rango?: {termometro.dentro_rango()}")  # True, 25°C está dentro del rango

termometro_f = Termometro(77, "Fahrenheit", "TermoPro", (-50, 150), 1)  # Instancia en Fahrenheit
print(f"Temperatura en Celsius: {termometro_f.a_celsius()}°C")  # 77°F -> 25.0°C

# Uso de métodos estáticos sin necesidad de instanciar
print(Termometro.fahrenheit_a_celsius(100))  # 100°F -> 37.78°C
print(Termometro.celsius_a_fahrenheit(0))    # 0°C -> 32.0°F

2.12.4 Métodos de clase

Supongamos que tenemos una clase Empleado, y queremos poder crear instancias de esta clase usando diferentes formatos de entrada, por ejemplo, a partir de una cadena de texto con información sobre el empleado.

class Empleado:

    def __init__(self, nombre, apellido, sueldo):
        self.nombre = nombre
        self.apellido = apellido
        self.sueldo = sueldo

    # Método de instancia: Muestra la información del empleado
    def mostrar_info(self):
        return f"{self.nombre} {self.apellido} - Sueldo: {self.sueldo}"

    # Classmethod: Crea un empleado desde una cadena de texto con nombre, apellido y sueldo
    @classmethod
    def desde_string(cls, string_empleado):
        nombre, apellido, sueldo = string_empleado.split("-")
        return cls(nombre, apellido, float(sueldo))
# Uso de classmethod para crear instancias
empleado1 = Empleado.desde_string("Ana-Garcia-50000")
empleado2 = Empleado.desde_string("Luis-Perez-60000")

# Muestra la información de los empleados creados
print(empleado1.mostrar_info())  
print(empleado2.mostrar_info())

import math

class Cuadrado:

    def perimetro(self):
        return lado*4

    @perimetro.setter

        self.lado = nuevo_perimetro/4

2.17 Encapsulación

  • Una de las ventajas de la programación orientada a objetos es la encapsulación de información.
  • La encapsulación oculta los detalles de implementación y el estado de cada objeto.
  • Quien utilice la clase solo necesita conocer la interfaz que se muestra hacia el exterior.
  • La encapsulación en programación es fundamental para construir aplicaciones complejas.

2.18 Niveles de acceso

Para preservar la encapsulación, muchos lenguajes de programación establecen límites de acceso a sus atributos

  • Privado (__): Sólo se puede acceder al elemento desde dentro de la propia clase.
  • Protegido (_): Sólo se puede acceder al elemento desde la clase o subclases de ésta.
  • Público: El acceso se permite desde cualquier parte del código, dentro o fuera de la clase.

Python no proporciona palabras reservadas para esto.

Esto permite evitar que se manipulen los atributos de objetos desde el exterior, y sólo permiten su modificación de forma controlada, a través de métodos de la propia clase Python no proporciona palabras reservadas para esto, se utilizan triquiñuelas en la nomenclatura (name manglig).

Python no 3ene ningún mecanismo para restringir el acceso a los elementos de una clase. Lo que hace es poner trabas sintác3cas para dificultar el acceso a ciertos elementos, aunque realmente siempre se puede acceder a todos los elementos, no hay nada completamente privado. Se u?liza el convenio siguiente para denotar el nivel de acceso de un elemento: ⊲ Cuando el iden?ficador del elemento comienza con doble subrayado el elemento es priva- do.
